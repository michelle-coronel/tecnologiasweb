DEFINICIÓN:

JAVA fue diseñado por James Gosling considerado como el padre del lenguaje de programacion JAVA y fue desarrollado por la empresa Sun Microsystems que fue constituida en 1983 y luego adquirida el 27 de enero del año 2010 por la compañia Oracle, es un lenguaje de programación grandemente empleado para codificar aplicaciones web.

Ahora bien Spring fue desarrollado por Rod Johnson un framework del lenguaje de programación JAVA, es una herramienta que se origina con la intención de simplificar y facilitar la construcción de aplicaciones, para aligerar y adelantar el desarrollo de aplicaciones JAVA.

CARACTERÍSTICAS PRINCIPALES:

  1. La idea principal es que Spring se emplee como una plataforma para desarrollar en JAVA de código abierto, es por esto que se ha convertido en el framework más conocido para Java en el entorno empresarial.
  2. Es compatible con frameworks web como son: Sring WebFlux y Spring MVC, permitinedo el procesamiento de datos por lotes. Siendo un framework liviano debido a su implementación POJO (Plain Old Java Object). Spring framework no exige a la persona que programa heredar alguna clase ni a implementar ninguna interfaz.
  3. Spring es completamente modular, permite en gran medida la programación basada en MVC (modelo, vista, controlador) y una implementación rápida en base a Inyeccion de dependencias
  4. Spring se aprecia como el padre de los Frameworks de JAVA, por lo que da soporte a varios frameworks como son: Struts, Hibernate, JSF, EJB, etc.
  5. Crea código reutilizable, de alto rendimiento y liviano.Ya que su finalidad es agilizar manejar, estandarizar y resolver los problemas que puedan existir a lo largo del desarrollo.

VENTAJAS:

  • Proporciona modularidad de alto nivel, para realizar una separación muy clara de las diferentes tareas que realizará cada clase de nuestra aplicación.
  • Simplifica el acceso a los datos gracias a ORM (Object Relational Mapping). Se puede ejecutar dentro de un contenedor Web o fuera de él, en una aplicación Swing.
  • Evita la dependencia entre clases. Minimiza el código repetitivo.
  • Provee escabilidad de la aplicación sin necesidad de modifcar sus clases.
DESVENTAJAS:
  • No se puede determinar si un objeto ha sido bien inyectado más que en tiempo de ejecución.
  • El contenedor Spring no es ligero, es decir no es aconsejable que se emplee en aplicaciones de tiempo real o aplicaciones móviles.
  • La configuracion de Spring esta inflada, es decir que para cada servicio tenemos que configurarlo en un XML de configuración.
TIPO DE LICENCIA:

El tipo de licencia certificada de Spring es Apache License 2.0, de software libre, la cual permite al usuario la libertad de emplearlo para cualquier propósito que requiera

ANÁLISIS DE LA CURVA DE APRENDIZAJE

Al estudiar Spring como framework de Java, no se puede indicar que su curva de aprendizaje sea fácil, ya que Spring incluye conceptos parcialmente innovadores por que se necesita estar en constante actualización.

Para una persona que este familiarizado con alguno de los conceptos no se le resultara difícil, pero como ventaja para favorecer el desarrollo se emplean IDEs Open Source, como SpringIDE que nos apoyan en lo que concierne a los archivos de configuración y proyectar el flujo de la aplicación.

EJEMPLO DE USO

Puedo mencionar como ejemplo de uso a la empresa Amazon, en la actualidad es la segunda empresa más famosa a nivel mundial, Amazon nació el 05 de Julio de 1994, en un garaje de Seattle, al principio empezó vendiendo libros en internet dirigida por Jeff Bezos, al indagar en un diccionario descubrió la palabra “Amazon”, que hace alusión al rio más grande del planeta, es por esto que decidieron colocar por nombre Amazon con el objetivo de crear la librería digital mas colosal a nivel mundial.

Amazon tiene algunos equipos y no tienen específicamente una única pila tecnológica, ya que la mayoría de los equipos emplean JAVA para el BackEnd, para guardar los datos se emplean DynamoDB y MySQL, por otro lado, para el Frontend se usan AngularJs y GWT.
