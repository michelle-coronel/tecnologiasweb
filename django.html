DEFINICIÓN:

Python fue creado por un programador holandés llamado Guido Van Rossum, en el año de 1991, es un lenguaje de programación multiuso, dicho lenguaje ayuda a los programadores de BackEnd para que escriban sus lineas de código logicas y claras.

Ahora bien Django fue nombrado en alusión al guitarrista de jazz gitano Django Reinhardt, es un framework de desarrollo web que se fundamenta en Python, ya que el mismo suministra las herramientas necesarias para la edificación de aplicaciones escalables y confiables.

CARACTERÍSTICAS PRINCIPALES:

  1. Es veloz
  2. Es bien cargado
  3. Es confiable
  4. Es escalable
  5. Es versátil

Tabla Características Principales

Es veloz Si se necesita terminar un proyecto lo más pronto posible y reducir costes, en Django se puede cimentar una excelente aplicación en un tiempo estimado mínimo.
Es bien cargado Es decir, si queremos emplear alguna cosa solamente se debe adaptar a nuestros requerimientos, ya que por medio de los paquetes de Django se lo podrá localizar.
Es confiable Django ejecuta por defecto ciertas medidas de seguridad. En donde se ocupa en utilizar todo esto de forma elemental.
Es escalable Se pude escalar desde lo normal a una aplicación inmensa, con la garantía de que funcione rápidamente y sea duradero.
Es versátil Se puede emplear para guardar noticias por sitios web de prensa, blogs, etc. Es decir, para lo que necesitemos.

VENTAJAS:

  • Django tiene una gran gama de documentos en linea, por lo que los desarrolladores encuentran facilmente las soluciones.
  • Trabaja bajo el patrón MVC (modelo, vista, controlador), lo que posibilita un desarrollo ágil y reutilizable.
  • Sostiene múltiples motores de bases de datos y son compatibles con PostgreSQL, MySQL, SQLite, etc.
  • Contiene un ORM que se encarga de la comunicación con las bases de datos, admite consultas avanzadas y puede optimizarse.
DESVENTAJAS:
  • Documentación extensa.
  • Overpack.
  • Rendimiento.
  • Plantilla.
Documentación extensa A veces puede llegar a ser incomprensible. No es apto para proyectos pequeños.
Overpack Se han encapsulado diversos métodos y clases, en donde se emplean directamente, pero resulta difícil modificarlos.
Rendimiento Su rendimiento es bajo a comparación con otros frameworks.
Plantila Dicha plantilla ejecuta el codigo y el estilo del todo separados y la flexibilidad puede que no sea suficiente para los desarrolladores.
TIPO DE LICENCIA:

El tipo de licencia de Django es Open Source con licencia BSD (Berkeley Software Distribution), siendo una licencia de software simple, libre y abierto, como la licencia OpenSSL o MIT

ANÁLISIS DE LA CURVA DE APRENDIZAJE

Al emplear el lenguaje de programación Python, se debe aprender muchas cosas a la vez, esto varía dependiendo de cada persona y de su tiempo, como se pudo indagar Python y Django son un extraordinario dúo, ya que empresas famosas a nivel mundial lo usan, es por esto que Django se emplea para el rápido desarrollo de apps y aplicaciones web siendo una de las mejores opciones, siendo la curva de aprendizaje buena empezando desde estudiar las sintaxis y la naturaleza de Python para que así no resulte complejo desarrollar en Django.

Se debe tomar el tiempo para predominar Python, para asi comprender el soporte de los sistemas para el BackEnd con la finalidad de adaptarse a la terminología de los servidores.

EJEMPLO DE USO

Puedo citar como ejemplo de uso, una empresa que utiliza Django es Instagram.

Django se centra principalmente en hacer aplicaciones web que sean netamente rápidas y escalables.

Instagram de origen estadounidense, actualmente es propiedad de Meta, sus creadores son: Kevin Systrom y Mike Krieger, es una red social que fue lanzada el 06 de Octubre del 2010 siendo una aplicación muy reconocida a nivel mundial, por lo que las tecnologías web juegan un papel fundamental en el mundo, ganando fama de manera exponencial por lo que llego a tener más de 100 millones de usuarios activos en el año 2012 y mas de 300 millones en el año 2014, y en la actualidad aquellas cifras siguen ascendiendo.

Ahora bien, Instagram emplea Django, que es un framework de Python, siendo un gran marco web de codigo abierto, por lo que ha permitido a los desarrolladores crear diversas paginas web, aplicaciones web, servicios web, etc.

Como sabemos Django está proyectado a apoyar a los desarrolladores para que instauren una aplicación de manera ágil y segura.

Kevin Systrom, codificó un prototipo HTML denominado burbn (versión inicial de Instagram), en donde su eje principal fue en compartir fotos, un dia se encontró con inversores a lo que les mostró burbn y fue un triunfo ya que sus inversores le financiaron con $500.000.

Aunque cabe recalcar que Instagram empleo un paquete de lenguajes los cuales son:

Parte del BackEnd:

  1. Django
  2. Celery
  3. Redis Cassandra
  4. RabbitMQ
  5. PostgreSQL
  6. Amazon S3 (Almacenar imágenes)
  7. Node.js (Para notificaciones push)

Por ejemplo, para indicar fallos en Instagram, utilizan Sentry que es una aplicación que se deriva de Django de open Source en donde Django ORM gestiona una base de datos que soporta millones de usuarios.

Siendo Django un magnífico marco web, en donde los ingenieros de Instagram apuestan que el mejor lenguaje es Python ya que permite gestionar un alto volumen de datos o visitas que Instagram recibe diariamente
